# 🏭 산업체 제조 환경 기반 문제 해결 AI Agent

> 산업체 제조 환경을 가정한 **반도체 결함 판별 AI Agent**를 개발하는 해커톤 프로젝트 🚀

라벨이 없는 예시 이미지를 기반으로, 트랜지스터에 **결함이 존재하는지 여부(0/1)** 를 자동 판단하는 에이전트를 설계·구현하는 것을 목표로 합니다.

## 1. 기획 배경 🧠

### 산업 현장의 현실적인 문제 ⚙️

반도체 제조 공정에는 다음과 같은 문제가 반복적으로 발생합니다.

- **결함 데이터의 라벨링 비용이 매우 큼** 💸
- 신규 공정 / 신규 장비 도입 시  
  → **충분한 학습 데이터 확보가 어려움** 📉
- 검사 기준이 사람·시점에 따라 달라질 수 있음
- 실제 현장에서는  
  **“완벽한 모델”보다 “일관된 판단 시스템”이 더 중요** ✅

해당 프로젝트는 이러한 현실을 반영하여,  
**소량의 무라벨 데이터만 주어진 상황에서 어떻게 신뢰 가능한 판단을 자동화할 것인가**를 핵심 문제로 삼습니다.

---

## 2. 문제 정의 🎯

### 주어진 조건 📌
- 라벨이 없는 **예시 이미지 20장**
- 실제 평가 대상은 **100장 이상의 이미지**
- 이미지에는 트랜지스터 구조가 포함되어 있으며,
  - 결함이 존재하거나
  - 정상 상태일 수 있음

### 해결해야 할 문제 ❓

> **무라벨 환경에서 이미지 기반으로 결함 유무를 판단하고,  
> 판단 결과를 스스로 검증·보정하며,  
> 최종 결과를 자동으로 제출하는 AI Agent를 설계하라.**

---

## 3. 접근 전략 개요 🧩

본 프로젝트는 단일 모델의 성능 극대화보다,  
**에이전트 관점의 문제 해결 구조**를 설계하는 데 초점을 둡니다.

### 핵심 아이디어 💡

- 한 번의 판단으로 끝내지 않는다
- 판단 과정 자체를 **검증하고 보정**한다 🔍
- 결과가 이상할 경우 **정책적으로 재확인**한다 🔁
- 전체 과정은 **완전 자동화**한다 🤖

---

## 4. 활용 로직 / Agent 설계 🧱

본 에이전트는 멀티모달 LLM + 규칙 기반 판단 로직을 결합해, **설명 가능한 결함 판별**을 목표로 합니다.

### 관찰 기준(결함 시그널) 🔍

- 리드 결손/소실/짧음: 리드가 3개가 아니거나 일부가 사라진 형태
- 리드 비정상 결합: 리드가 정상 위치에 꽂혀 있지 않거나, 끝단이 떠 있는 형태
- 리드 휨/교차/접촉: 리드가 과도하게 휘거나 서로 맞닿은 형태
- 소자 기울기/회전: 정상 대비 소자가 눈에 띄게 기울어진 형태
- 정렬 이상(위치 이탈): 배경 패턴 기준으로 중심이 크게 벗어난 형태 (단독 판단은 보류)
- 패키지 파손/크랙: 외곽 깨짐/뜯김이 확인되는 형태

### 판단 규칙 설계 ⚖️

결함 신호의 위험도를 반영해, 아래 2단계 규칙을 사용합니다.

Hard Rule(즉시 불량)
- 제조 관점에서 치명적인 결함(예: 리드 소실/심한 휨·접촉/패키지 파손)은 관찰 시 즉시 비정상(1) 으로 판정합니다.

Soft Rule(조합/가중 판단)
- 단독으로는 애매한 신호(예: 약한 기울기, 정렬 이탈)는 다른 신호와의 조합 및 확신도를 함께 고려해 판정합니다.
- 예) 기울기 + 정렬 이탈처럼 함께 나타나면 불량 가능성을 강화

### 불확실성 처리(검증/보정) 🔁

- 현장 데이터는 애매한 케이스가 많기 때문에, 에이전트는 다음 정책으로 불확실성을 관리합니다.
- 애매한 케이스는 재확인(재관찰) 하여 판단을 보정
- 전체 결과 분포가 비정상적으로 쏠리면(예: 전부 0 또는 전부 1) 정책적으로 재검증
- 이를 통해 “모델의 순간 판단”이 아니라 안정적인 최종 판정을 목표로 합니다.
